# Gamma → Chatium: мобильная адаптация

Десктоп с Gamma переносим **буквально 1:1** ([[gamma-to-chatium-transfer]]). Мобильную версию делаем **заново как UI/UX-дизайнер**: тот же визуальный язык (палитра, шрифты, акценты), но цель — **читаемость и красота**, а не точное совпадение.

## Почему НЕ ресайлить DOM Gamma под мобилку

Родная мобильная вёрстка Gamma в статике ужасна: на 390px она сохраняет десктопные многоколоночные раскладки → hero ломается (заголовок переносится по слогам), документ ~17–18 тыс. px, всё узко и нечитаемо.

Ресайлить экспортированный DOM почти невозможно чисто: вложенные `display:grid` + `display:contents`-ячейки, `themed-shape-outside-text`-гриды, accent-маски, хеш-классы Emotion и **бистабильный height-chain** (см. transfer-skill). Любая правка раскладки рискует уронить layout, а «красиво» из этого не сделать.

**Проверено эмпирически:** generic `@media`-ресайл DOM Gamma (force `grid-template-columns:1fr`, `card-layout` в колонку и т.п.) даёт кривой результат — колонки частично остаются рядом, страница ~2–3× выше нужного, «squeezed Gamma», а не редизайн. Поэтому — отдельный слой ниже.

**Решение: отдельный, самодостаточный мобильный слой**, переключаемый `@media`:
- десктоп (`>=769px`): показываем клон Gamma, мобильный слой `display:none`;
- мобилка (`<=768px`): клон `display:none`, показываем мобильный слой.

Каждая раскладка изолирована → клон не дестабилизируется, а мобилку лепим с нуля и красиво.

## Шаги

### 1. Обернуть и переключать

В `PAGE_HTML`:

```
<body class="chakra-ui-light">
  <div class="a475-desktop"> {клон .doc-root} </div>
  <div class="a475-mobile">  {рукотворный мобильный HTML} </div>
  <script> reflow-nudge </script>
</body>
```

Переключение — **в отдельном `<style id="a475-mobile-css">`** (НЕ в `#a475-overrides`, который дёргает reflow-nudge; иначе правила сотрутся при pulse):

```css
.a475-mobile { display: none; }
@media (max-width: 768px) {
  .a475-desktop { display: none !important; }
  .a475-mobile  { display: block !important; }
}
```

**Огради reflow-nudge десктопом**: `function fix(n){ if (window.innerWidth<=768) return; ... }` — иначе на мобилке скрипт бесконечно «чинит» скрытый (offsetHeight=0) клон.

### 2. Извлечь контент из клона (структурированно)

**Критично:** в Gamma **нет тегов `<h1>..<h4>`** — заголовки это `div[data-node-view-content-inner="heading"]` (и один `"title"`). Запрос `querySelectorAll('h1,h2,h3')` вернёт пусто! Типы контента — по атрибуту `data-node-view-content-inner`: `title`/`heading` → заголовок, `paragraph` → абзац, `bullet` → пункт, `button` → кнопка. Колонки секций — `gridCell` / `smartLayoutCell` внутри `gridLayout`/`smartLayout`.

Извлекай **с живого клона** одним `browser_evaluate (filename=content-<id>.json)`: по каждой `.card-wrapper` собери `{ heading[], intro[], cells:[{heading,body,bullets,img}], buttons[], imgs[] }`. Брать только «листовые» content-узлы (`!el.querySelector('[data-node-view-content-inner]')`), top-level отделять от тех, что внутри ячеек (`closest(cell)`). Картинки — URL imgproxy/cdn.gamma.app, **оставляем как есть**. (Готовый экстрактор — в `capture-page` шаге transfer-skill и инлайн в build-конвейере.)

### 2b. Рендер мобилки (готовый генератор)

Структуру → мобильный HTML собирает **`render-mobile.cjs`** (в корне общей папки) по эвристикам формы NESO/Gamma: card[0] → hero (лого + главная картинка + lead + кнопки), последняя карта с кнопкой → CTA (формат + цена `solid`-callout), карта с ≥2 текстовыми ячейками → стек карточек (с картинкой ячейки → media-card), карта «текст + картинка-ячейка» → banner+абзацы, абзацы вида `— Имя` → атрибуция автора. Компоненты и CSS — в **`gamma-mobile-shared.cjs`** (scoped под `.gamma-mobile`). Тогда `<id>-mobile.cjs` = одна строка: `module.exports={MOBILE_HTML:require('./render-mobile.cjs').render(require('./content-<id>.json'))}`.

Картинка hero: **исключай лого** (`Logotip`/`/original/`), бери `/generated-images/`; лого подавай отдельно как маленький `gm-logo`.

### 3. Палитра и шрифты (брать из клона)

Снять `getComputedStyle` с клона:
- body: `Inter`, ~18px, цвет `rgb(44,41,38)` (#2c2926), line-height 1.6;
- акцент (золото): `rgb(231,191,106)` = `#e7bf6a`; глубокое золото для текста на светлом — ~`#b98b3e`;
- кнопка: `linear-gradient(45deg,#da9b49,#f3e38a)`, тёмный текст;
- callout: `rgba(231,191,106,.75)`; фон страницы: `rgb(245,245,246)`;
- заголовки — serif. На десктопе Gamma заголовки визуально serif, т.к. heading-font `Bricolage Grotesque` **не содержит кириллицы** и падает в системный serif. В мобильном слое используем стек `"Bricolage Grotesque", Georgia, "Times New Roman", serif` — кириллица так же уходит в Georgia/serif → совпадает по духу.

**Акцент — per-page!** У разных страниц своя тема: золото `#e7bf6a`, но встречается красный `#DA1B2E` и др. Мобилка должна совпадать с десктопом по цвету. `gamma-mobile-shared.cjs` параметризован переменными (`--gm-accent`, `--gm-accent-rgb`, `--gm-accent-deep`, `--gm-btn-grad`, `--gm-btn-text`, `--gm-on-accent`; дефолт — золото). `build-gamma.cjs` детектит инлайновый `--accent-color` из захваченного HTML и, если он не золотой, дописывает в `#gamma-mobile-css` override этих переменных: считает rgb, тёмный оттенок (`×0.78`), градиент кнопки и **цвет текста на кнопке по яркости** (тёмный для светлого золота, белый для тёмных/насыщенных). Так красная страница получает красную мобилку автоматически.

### 4. Сверстать мобильный слой (один столбец)

Всё под префиксом-классом (`.a475-mobile` / `.a475-m-*`), чтобы захваченный CSS Gamma не протекал. Компоненты, проверенные на a475:
- **hero**: лого по центру, крупный serif-заголовок, lead, золотая CTA-кнопка, картинка-баннер (`border-radius:22px; object-fit:cover; height~300px; box-shadow`);
- **секция**: H2 + lead + контент; разделители `border-top` между секциями;
- **media-card**: картинка сверху (height~190px) + заголовок + текст;
- **numbered-card**: золотой кружок с цифрой + заголовок + текст;
- **callout**: золотой полупрозрачный блок; вариант `solid` (золотой) для «Миссия Дракона» с буллетами; вариант `green` для позитивных пометок;
- **step**: кружок-цифра + заголовок + текст (для пошаговых блоков);
- **CTA**: full-width кнопка.

Хранить как отдельный модуль (`a475-mobile.cjs`, экспорт `MOBILE_CSS`/`MOBILE_HTML`) и `require` в build-скрипте — чтобы build оставался читаемым.

### 5. Сборка, sync, проверка

Пересобрать `pageHtml.ts`, синкнуть (см. transfer-skill, `--apply --no-new --allow-mixed-create-delete`). Проверить в браузере:
- мобилка (`browser_resize` 390×844, `?v=N`): один столбец, hero читаемый, картинки на месте, высота резко меньше родной (для a475 ~8.9k px vs 17.9k родных), отступы/типографика комфортные;
- **десктоп не сломан** (`resize` 1440): свежая загрузка → `.card-wrapper` offsetHeight=512, `body`=6353, `.a475-mobile` display:none, `.a475-desktop` display:block.

## Подводные камни

- Правила переключения и мобильный CSS — **только** в `#a475-mobile-css`, не в `#a475-overrides`.
- Обязательно огради reflow-nudge `innerWidth>768`.
- Не полагайся на наследование от захваченного CSS — задавай значения явно (`box-sizing`, цвета, размеры) и скопь под `.a475-mobile`.
- Картинки — те же ссылки на Gamma (imgproxy/cdn); не перезаливай.
- Точку перелома (768px) держи одинаковой в переключателе и в `@media` мобильного CSS.
