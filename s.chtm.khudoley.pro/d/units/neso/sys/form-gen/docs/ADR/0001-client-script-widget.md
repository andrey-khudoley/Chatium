# ADR-0001: Встраивание форм клиентским `<script>`-виджетом, конфигурация форм в Heap

Дата: 23-07-2026. Статус: принято.

## Контекст

Первая редакция §5.1 спеки (утро 23-07-2026) решала встраивание **серверным импортом**: страница воркспейса импортирует `attachOrderForm(ctx, PAGE_HTML, config)`, фрагмент формы инъецируется в HTML на сервере, конфиг формы живёт в коде страницы (параметры вызова), в Heap конфигурации форм не хранятся. Клиентский `<script>`-виджет был рассмотрен и отвергнут как основной путь («лишний сетевой цикл, голый экран до загрузки»), с оговоркой «остаётся запасным для страниц вне воркспейса».

В тот же день требование расширилось: формы должны подключаться **на любых страницах** — не только клонах Gamma `p/units/neso/pages/aNNN/`, но и на прочих страницах аккаунта и на **внешних доменах**. У внешней страницы нет серверного кода воркспейса: импортировать функцию некуда, инъецировать HTML некому.

## Решение

1. **Единственный механизм подключения — клиентский виджет:** `<script src=".../widget.js?form=<formID>">`. Два режима отображения: inline (скрипт заменяет якорь `<div id="<formID>">` формой в потоке страницы) и модалка (скрипт регистрирует обработчик вызова модального окна).
2. **Форма — сущность с идентификатором `formID`, конфигурация — в Heap** (таблица форм): предложение GC, заголовок, тексты. Генератор админ-страницы создаёт запись и выдаёт сниппеты подключения. Правка формы — в админке, страницы не трогаются.
3. **Внешние домены поддерживаются:** эндпоинт отправки отвечает с `Access-Control-Allow-Origin: *` (whitelist доменов отвергнут: CORS не защищает сервер, боты шлют POST напрямую; эндпоинт публичный, без credentials — `*` безопасен); раздача скрипта CORS не требует. Защита от спам-сабмитов — отдельно (rate-limit/honeypot, волна 3).

## Последствия

- Прежний план «конфиг в коде страницы, таблиц форм нет» отменён: таблица форм становится второй Heap-таблицей модуля, её схема — необратимое решение (гейт волны 2, прототип на временных id).
- Принятая цена клиентского виджета: форма появляется после загрузки скрипта (для inline-режима возможен пустой якорь до инициализации).
- Контракт виджета (способ доставки конфига, API модалки, формат `formID`) прорабатывается прототипом волны 1 и замораживается с волны 2 как внешний контракт.
- Для страниц `aNNN` подключение упростилось до вставки строки `<script>` через `index.tsx` (регенерация `pageHtml.ts` из Gamma её не задевает).
- Отменённые альтернативы: серверный импорт `attachOrderForm` (работает только внутри воркспейса), `app.function` как канал подключения (та же причина), конфиг в параметрах `src` (размазывает конфиг по страницам, правка формы = правка страниц).
