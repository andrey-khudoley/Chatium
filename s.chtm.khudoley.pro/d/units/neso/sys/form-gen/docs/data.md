# Данные

Хранилище модуля form-gen — две Heap-таблицы (волна 1). Id таблиц зафиксированы спекой ([`docs/spec/spec.md`](spec/spec.md) §3) и объявлены каждый ровно в одном файле аккаунта; сегмент `__stage_` — dev-копия, при переносе `/to-prod` трансформируется в `__prod_` (правило переноса — корневой `CLAUDE.md` §0).

## Таблицы

### FormsTable — конфигурации форм

Файл: [`tables/forms.table.ts`](../tables/forms.table.ts).
Id: `t__form-gen__forms__stage_2xnqat`.

| Поле | Тип | Индексация | Назначение |
|------|-----|------------|------------|
| `slug` | `Heap.String` | плоское, корневое — единственный горячий фильтр | formID формы (слаг nanoid с префиксом `fg`, см. `SLUG_PREFIX` в `config/constants.ts`); по нему виджет-роут (`widget?form=<slug>`) и `/api/submit` находят форму; формат гарантирует валидный DOM-id/CSS-селектор |
| `offers` | `Heap.Any` | нет | JSON-массив предложений `{ offerId, title, price, currency }` — снимок отображения для мультиоффера (форма может предлагать выбор из нескольких офферов) |
| `appearance` | `Heap.Any` | нет | Настройки внешнего вида формы; контракт поля открыт, откладывается на волну 2 (MVP). Дефолт при пустом значении — `DEFAULT_APPEARANCE` в `config/constants.ts` |

Системные поля `id`/`createdAt`/`updatedAt` — автоматические поля Heap, в схеме не объявляются.

Реестр «форма → страница» модуль не ведёт: связь формы со страницей, где она встроена, снаружи Heap (сниппет `<script>`/`<div>` внедряется вручную в разметку целевой страницы).

### FormGenSettings — настройки подключения GetCourse

Файл: [`tables/settings.table.ts`](../tables/settings.table.ts).
Id: `t__form-gen__settings__stage_x2plui`.

Key-value таблица по образцу `d/system/broker/tables/settings.table.ts`.

| Поле | Тип | Индексация | Назначение |
|------|-----|------------|------------|
| `key` | `Heap.String` | плоское | имя настройки |
| `value` | `Heap.Any` | нет | значение настройки (произвольная форма) |

Ключи волны 1 (константы `GC_SETTINGS_KEYS` в [`config/constants.ts`](../config/constants.ts)):

| Ключ | Назначение |
|------|------------|
| `gc_school_url` | URL школы GetCourse |
| `gc_school_key` | ключ школы GC (авторизация Legacy-импорта) |
| `gc_developer_key` | ключ разработчика GC — только сохраняется в волне 1, кодом не читается |

## Окружения stage/prod

Записи настроек — данные окружения, а не код: `/to-prod` их **не переносит**. Каждое окружение заполняет свою таблицу настроек отдельно через свою админку (`web/admin/` + `pages/admin/AdminPage.vue`):

- dev-копия работает с тестовой школой GC (полигон прототипа);
- prod-копия — с боевой школой GC.

Id самих таблиц (`FormsTable`, `FormGenSettings`) переносятся кодом как обычно — трансформация сегмента `__stage_` → `__prod_` при `/to-prod`.

## Что НЕ хранится

Заявка/заказ посетителя в Heap модуля не хранится (спека §2): `/api/submit` создаёт заказ сразу в GC через Legacy-импорт, у себя данные заказа не сохраняет. Единственный след — логи (`ctx.account.log()`); событие в брокере наблюдаемости — волна 3.
