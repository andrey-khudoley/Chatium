# Проект form-gen

## Что это

Генератор форм заказа: клиентский `<script>`-виджет для страниц neso и внешних сайтов, наполняемых предложениями GetCourse. Прототип волны 1 (`GET /widget?form=<slug>` + `POST /api/submit`) реализован.

## Где искать детали

- Источник истины: `docs/spec/spec.md` — код меняется только после того, как изменение описано здесь (spec-as-source)
- Техническая информация и changelog: `README.md`
- Архитектура: `docs/architecture.md`
- API: `docs/api.md`
- Данные: `docs/data.md`
- Решения: `docs/ADR/` (0001 — разворот на клиентский виджет)
- История диалогов: `docs/LLM/`

## Ключевые точки кода

- Виджет-роут `widget/index.ts` — отдаёт запечённый JS по `GET /widget?form=<slug>`
- Публичный приём заявок `api/submit.ts` — `POST`, `x-www-form-urlencoded`, CORS `*`, прямой вызов GetCourse
- Временный GC-клиент `lib/gc/*` — дубль Legacy-импорта, подлежит удалению в MVP
- Админка `web/admin/index.tsx` + `pages/admin/AdminPage.vue`
- Heap-таблицы `tables/forms.table.ts`, `tables/settings.table.ts`

## Ключевые правила

- **spec-as-source.** Любое изменение поведения, контрактов, данных или структуры — сначала правка `docs/spec/spec.md`, потом код.
- Не хардкодить URL — только `config/routes.tsx` / `withProjectRoot`.
