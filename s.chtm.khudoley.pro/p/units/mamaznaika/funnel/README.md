# p/units/mamaznaika/funnel — сайт проекта «Мама Знайка · вебинарная воронка»

Одностраничный Chatium-сайт (SSR Vue) для клиента Александра Миляева (АНО ДПО СОЦ
«Мама Знайка»): рабочее пространство discovery-стадии по обновлению вебинарной
воронки. Четыре вкладки: **Главный документ** (это НЕ КП), **Текущая архитектура**,
**Ожидаемая архитектура**, **Вопросы**. Любой посетитель (в т.ч. анонимный) может
оставить ответ на вопрос без ввода имени — ответ сохраняется как «комментарий» и
появляется под вопросом.

Сайт одноязычный (русский). Собран по образцу `p/units/ipsen/kp`, но без
мультиязычности и с другим набором вкладок.

## Текущее состояние

**Каркас.** Структура, вёрстка и механика ответов готовы. Контент разделов — заглушки
(часть с реальными фактами из переписки, часть — заблокированные разделы
«заполнится по мере аудита»). Наполняется позже.

## Структура

```
mamaznaika/funnel/
├─ index.tsx               # app.html('/') — SSR: читает ответы из Heap, рендерит SitePage
├─ pages/SitePage.vue      # весь сайт: 4 вкладки, формы ответов под вопросами
├─ api/answers/create.ts   # POST // @shared-route: приём ответа (requireAnyUser — аноним ok)
├─ api/answers/all.ts      # GET: публичная выгрузка всех вопросов+ответов одним JSON
├─ tables/answers.table.ts # Heap-таблица ответов, привязка по questionId
├─ shared/content.ts       # весь контент (CONTENT: SiteContent, RU), id вопросов  // @shared
├─ shared/styles.ts        # дизайн-CSS (светлая тема, красный акцент)
├─ config/routes.tsx       # PROJECT_ROOT + реестр путей  // @shared
└─ манифесты: .dir.json .workspace.json package.json tsconfig.json jsx.d.ts vue-shim.d.ts
```

## Как это работает

1. `index.tsx` на сервере читает `Answers.findAll` (Heap только на сервере),
   группирует ответы по `questionId` и передаёт SSR-пропсом в `SitePage.vue`.
   Весь статический контент компонент берёт сам из `shared/content.ts`.
2. Контент — единый объект `CONTENT: SiteContent` (русский). Для мультиязычности
   позже — обернуть в `Record<Lang, SiteContent>` по образцу ipsen/kp.
3. Вкладки: `doc` (Главный документ, дефолтная) / `current` (Текущая архитектура) /
   `target` (Ожидаемая архитектура) / `questions` (Вопросы). Обе подписи вкладки
   (полная/короткая) в кнопке, нужную показывает CSS по ширине экрана.
4. Форма ответа — только текст, без поля имени. Отправляет через
   `apiCreateAnswerRoute.run(ctx, { questionId, text })` (роут `// @shared-route`).
5. `api/answers/create.ts` вызывает `requireAnyUser(ctx)` (аноним разрешён),
   валидирует текст/вопрос и пишет в `Answers`. У реального пользователя берётся
   `displayName`, у анонима имя пустое — клиент показывает локализованный бейдж.
6. Главный документ: разделы 1–2 заполнены реальным контекстом из переписки,
   разделы 3–7 визуально заблокированы (`locked: true`, пунктирная рамка + замок).

## Данные

Таблица `Answers` (`t__mamaznaika-funnel__answers__mz9Fk3p`): `questionId`, `text`,
`authorName` (пусто для анонимов), `authorUserId`, `authorType`, `createdAtMs`.
Системные `id/createdAt/updatedAt` — Heap автоматически. Стабильные id вопросов
(`q1`…`q6`) — в `shared/content.ts`.
