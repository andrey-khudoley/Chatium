# raffle07 — «Розыгрыш призов» (Ларина)

Standalone-страница анимированного розыгрыша двух призов среди реального списка участников.
Порт дизайна Claude Design «Розыгрыш призов.dc.html» на self-contained vanilla-JS страницу.

**URL:** `/p/units/larina/raffle07`

## Текущее состояние

- ✅ Один серверный роут `index.tsx` (`app.get('/')`) отдаёт готовый HTML из `pageHtml.ts`.
- ✅ 42 реальных участника захардкожены в `USERS` (email замаскированы).
- ✅ Победители: `WINNER_INDICES = [15, 25]` — Галина Берёзкина (1 место), Ольга Самсонова (2 место).
- ✅ Двухэтапная анимация розыгрыша (отбор финалистов по трём столбцам → финал среди троих),
  конфетти на canvas, reveal- и results-оверлеи — воспроизведены 1:1 с дизайном.
- ✅ Мобильная адаптация (`@media 900px/560px`): сетка участников всегда 3 столбца, ужатые ячейки,
  скрытие email на узких, слоты в ряд, скроллящиеся оверлеи.

## Структура

| Файл             | Назначение                                                             |
| ---------------- | ---------------------------------------------------------------------- |
| `index.tsx`      | Точка входа. Роут `/`, отдаёт `PAGE_HTML` через `ctx.resp.html`.        |
| `pageHtml.ts`    | Вся страница: HTML + CSS + клиентский скрипт + данные (`USERS`, призы). |
| `tsconfig.json`  | Конфиг TypeScript проекта.                                             |
| `.CHATIUM-LLM.md`| Указатель для LLM.                                                      |

## Как поменять

- **Победители** — `WINNER_INDICES` в `pageHtml.ts` (два 0-based индекса в `USERS`).
- **Призы** — `PRIZES` (1/2 место) и `CONSOLATION` в `pageHtml.ts`.
- **Участники** — массив `USERS` в `pageHtml.ts`.
- **Тайминги/визуал** — соответствующие функции в клиентском скрипте `pageHtml.ts`.

## Особенности реализации

- Дизайн-исходник на фреймворке Claude Design (DCLogic = React + support.js). Здесь переписан
  на чистом vanilla JS: состояние + императивный рендер ячеек/слотов/оверлеев, без внешних зависимостей.
- Серверных данных нет: heap-таблицы, API, авторизация не используются.
- email участников замаскированы по схеме дизайна — полные адреса в исходник страницы не попадают.
