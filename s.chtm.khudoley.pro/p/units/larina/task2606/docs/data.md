# Данные — task2606 (диагностика GetCourse)

Проект использует две Heap-таблицы: одну для хранения обращений с внешних GetCourse-страниц и одну для хранения настроек. Обе таблицы работают без реляционных связей друг с другом — это плоские key/value и event-log хранилища.

---

## Таблицы

### Diagnostics — обращения диагностики

**UID:** `t__larina__task2606__diagnostics__<rnd>`

Каждый визит внешней GetCourse-страницы порождает одну запись. Дедупликации нет — записи накапливаются как event-log.

| Поле        | Тип Heap                       | Описание                                                                                                                                                                                                                                                                           |
| ----------- | ------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `id`        | системное                      | PK, генерируется Heap автоматически                                                                                                                                                                                                                                                |
| `createdAt` | системное                      | дата создания, используется для сортировки                                                                                                                                                                                                                                         |
| `updatedAt` | системное                      | дата обновления                                                                                                                                                                                                                                                                    |
| `visitorId` | `Optional(String)`             | ID визитёра из `window.accountUserId` на GetCourse-странице (не Heap PK)                                                                                                                                                                                                           |
| `ip`        | `Optional(String)`             | IP визитёра из `window.requestIp`                                                                                                                                                                                                                                                  |
| `url`       | `Optional(String, searchable)` | URL посещённой страницы — только origin+pathname, без query. Поле searchable                                                                                                                                                                                                       |
| `params`    | `Optional(String)`             | query-строка из URL (`location.search`), например `?a=1&b=2`                                                                                                                                                                                                                       |
| `dom`       | `Optional(String)`             | outerHTML страницы без тегов `<script>`, `<style>` и HTML-комментариев. Без ограничения размера. В листинг админки не выводится — только в детальный просмотр                                                                                                                      |
| `info`      | `Optional(Any)`                | плоский JSON прочих window-переменных: `isSublogined`, `accountId`, `gcSessionId`, `controllerId`, `actionId`, `nowTime`, `dateOffset`, развёрнутый `userInfo` (ключи вида `userInfo.isAdmin`, `userInfo.canCms` и т.д.). Поля `id`, `ip`, `url`, `params` в `info` не дублируются |

**Сортировка** в запросах: `order: [{ createdAt: 'desc' }]`.

**Фильтры в админке:** `visitorId`, `ip`, `url` — точное совпадение; `url` дополнительно поддерживает поиск через searchable-индекс.

### Settings — настройки (key/value)

**UID:** `t__larina__task2606__setting__<rnd>`

Хранилище конфигурации проекта в виде пар ключ/значение.

| Поле        | Тип Heap             | Описание           |
| ----------- | -------------------- | ------------------ |
| `id`        | системное            | PK                 |
| `createdAt` | системное            | дата создания      |
| `updatedAt` | системное            | дата обновления    |
| `key`       | `String(searchable)` | имя настройки      |
| `value`     | `Any`                | значение настройки |

**Известные ключи:**

| Ключ                  | Тип значения | Дефолт | Семантика                                                                               |
| --------------------- | ------------ | ------ | --------------------------------------------------------------------------------------- |
| `diagnostics_enabled` | boolean      | `true` | Тумблер приёма обращений. При `false` ingest-обработчик не создаёт записи в Diagnostics |

**Запись** выполняется через upsert под `runWithExclusiveLock` — защита от гонки при параллельных запросах.

---

## Поток записи (ingest)

1. Внешняя GetCourse-страница отправляет POST-запрос на ingest-эндпоинт.
2. Обработчик читает `diagnostics_enabled` из Settings.
3. Если `false` — запрос игнорируется, запись не создаётся.
4. Если `true` — создаётся новая запись в Diagnostics с полями `visitorId`, `ip`, `url`, `params`, `dom`, `info`.
5. `url` сохраняется без query-части; `params` — отдельно.
