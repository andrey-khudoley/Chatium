# API — task2606

API проекта состоит из двух частей: **публичные эндпоинты** (встраиваемый скрипт и приём снимков) и **административные эндпоинты** (просмотр записей диагностики и управление настройками). Публичные эндпоинты доступны без авторизации и предназначены для внешних страниц (GetCourse и аналоги). Административные — только для роли Admin.

---

## Эндпоинты

| Method | Path                            | File                              | Auth  | Назначение                                 |
| ------ | ------------------------------- | --------------------------------- | ----- | ------------------------------------------ |
| GET    | `/api/embed/script`             | `api/embed/script.ts`             | —     | Встраиваемый JS-скрипт для внешних страниц |
| POST   | `/api/ingest`                   | `api/ingest.ts`                   | —     | Приём снимка страницы от скрипта           |
| GET    | `/api/admin/diagnostics/list`   | `api/admin/diagnostics/list.ts`   | Admin | Листинг записей диагностики с фильтрами    |

| POST   | `/api/admin/settings/save`      | `api/admin/settings/save.ts`      | Admin | Сохранение настройки                       |

## Детали эндпоинтов

### GET `/api/embed/script`

Отдаёт встраиваемый JS-скрипт.

**Заголовки ответа:**

- `Content-Type: application/javascript; charset=utf-8`
- `Cache-Control: public, max-age=60` (короткий кэш — обновления скрипта доходят до встроенных страниц за ~1 мин)

**Использование на стороннем сайте:**

```html
<script src="https://s.chtm.khudoley.pro/p/units/larina/task2606/api/embed/script"></script>
```

Скрипт собирает снимок (visitorId, ip, url, params, dom, info) и отправляет его POST-запросом на `/api/ingest`.

**Момент захвата DOM:** не на `DOMContentLoaded`, а после события `window.load` и «успокоения» динамики — скрипт ждёт паузы в мутациях DOM (1.5 с без изменений) с жёстким потолком 12 с от момента load. Это нужно, потому что страницы GetCourse догружают контент (AJAX, React-виджеты) уже после `DOMContentLoaded`; ранний захват попадал бы только на скелет страницы. Запрос отправляется без `keepalive` (его лимит тела ~64 КБ обрезал бы крупный DOM); платформа ingest принимает тело как минимум до 6 МБ.

### POST `/api/ingest`

Кросс-доменный приём снимка. Платформа автоматически добавляет заголовок `access-control-allow-origin: *`.

**Тело запроса:** `Content-Type: text/plain`, строка JSON.

**Поля тела:**

| Поле        | Тип    | Обязательность | Описание                                                                      |
| ----------- | ------ | -------------- | ----------------------------------------------------------------------------- |
| `visitorId` | string | необязательно  | Идентификатор посетителя                                                      |
| `ip`        | string | необязательно  | IP-адрес                                                                      |
| `url`       | string | необязательно  | URL страницы                                                                  |
| `params`    | string | необязательно  | Параметры запроса                                                             |
| `dom`       | string | необязательно  | HTML-снимок страницы (без ограничения размера)                                |
| `info`      | object | необязательно  | Плоский объект с дополнительной информацией (вложенные объекты отбрасываются) |

Сервер принимает строку JSON (делает `JSON.parse`) или уже распарсенный объект. Все поля нормализуются к строке; поле `info` принимается только если это плоский объект.

**Ответы:**

| Условие                                                            | Ответ                                                |
| ------------------------------------------------------------------ | ---------------------------------------------------- |
| Запись сохранена                                                   | `{success: true}`                                    |
| Тумблер diagnostics_enabled выключен                               | `{success: false, message: "Диагностика отключена"}` |
| Пустой payload (visitorId, ip и url все пусты) или невалидный JSON | `{success: false}`                                   |

### GET `/api/admin/diagnostics/list`

Требует роль Admin (`requireAccountRole(ctx, 'Admin')`).

**Query-параметры:**

| Параметр    | Тип    | По умолчанию | Описание                                                       |
| ----------- | ------ | ------------ | -------------------------------------------------------------- |
| `limit`     | number | 20           | Число записей (диапазон 1..100)                                |
| `offset`    | number | 0            | Смещение (≥0)                                                  |
| `visitorId` | string | —            | Фильтр по visitorId (точное совпадение, пустой — игнорируется) |
| `ip`        | string | —            | Фильтр по ip (точное совпадение)                               |
| `url`       | string | —            | Фильтр по url (точное совпадение)                              |

Фильтры комбинируются по И. Сортировка по `createdAt` desc.

**Ответ:** `{success: true, rows: [{id, visitorId, ip, url, params, info, createdAt}], total, limit, offset}`

Поле `dom` в листинг не включается (хранится отдельно, запрашивается через `/get`).

**Query-параметры:**

### POST `/api/admin/diagnostics/delete`

Требует роль Admin. Удаляет одну запись диагностики по id.

**Тело запроса:** JSON `{id}`.

**Ответы:**

| Условие           | Ответ                                               |
| ----------------- | --------------------------------------------------- |
| Запись удалена    | `{success: true, id}`                               |
| `id` не передан   | `{success: false, error: "Параметр id обязателен"}` |
| Запись не найдена | `{success: false, error: "Запись не найдена"}`      |

В админке вызывается кнопкой «Удалить» в окне просмотра записи.

### POST `/api/admin/settings/save`

Требует роль Admin.

**Тело запроса:** JSON `{key, value}`.

Для ключа `diagnostics_enabled` значение нормализуется к boolean. Сохранение выполняется через upsert под `runWithExclusiveLock`.

**Ответ:** `{success: true, key, value}`

---

## Тумблер приёма

Настройка `diagnostics_enabled` (хранится в таблице настроек) управляет приёмом снимков на `/api/ingest`:

- **true** — снимки принимаются и записываются.
- **false** — `/api/ingest` отвечает `{success: false, message: "Диагностика отключена"}` без сохранения данных.

Значение изменяется через `POST /api/admin/settings/save` с `{key: "diagnostics_enabled", value: true/false}`.
