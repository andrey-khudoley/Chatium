# Larina Diagnostics (task2606)

Диагностический инструмент для внешних GetCourse-страниц. Внешний встраиваемый скрипт собирает снимок страницы (кто смотрит, откуда, что на экране) и сохраняет в Heap. Админка позволяет просматривать обращения, фильтровать их и управлять приёмом.

---

## Как встроить

На нужную страницу GetCourse добавить тег:

```html
<script src="https://s.chtm.khudoley.pro/p/units/larina/task2606/api/embed/script"></script>
```

Готовый тег с кнопкой копирования доступен на **главной странице** проекта: `https://s.chtm.khudoley.pro/p/units/larina/task2606/` — нажмите «Копировать» и вставьте сниппет в код нужной страницы.

Скрипт при загрузке страницы автоматически отправляет снимок на приём. При переносе проекта на другой домен — поменять хост в URL встройки.

---

## Что собирается

| Поле        | Источник                             | Описание                                                                                                      |
| ----------- | ------------------------------------ | ------------------------------------------------------------------------------------------------------------- |
| `visitorId` | `window.accountUserId`               | Идентификатор посетителя                                                                                      |
| `ip`        | `window.requestIp`                   | IP-адрес                                                                                                      |
| `url`       | `window.location`                    | URL страницы без query-параметров                                                                             |
| `params`    | `window.location.search`             | Query-параметры строкой                                                                                       |
| `dom`       | `document.documentElement.outerHTML` | Полный HTML без `<script>`, `<style>` и комментариев, без ограничения по размеру                              |
| `info`      | `window.*` (плоский JSON)            | `isSublogined`, `accountId`, `gcSessionId`, `controllerId`, `actionId`, `nowTime`, `dateOffset`, `userInfo.*` |

## Админка

URL: `https://s.chtm.khudoley.pro/p/units/larina/task2606/web/admin`

Доступна только для роли **Admin**.

Возможности:

- Лист обращений с пагинацией.
- Фильтры по `visitorId`, `ip`, `url` — по отдельности и в сочетании (точное совпадение).
- Разворачивание обращения: полные данные, `dom`, `info`.
- Удаление записи (кнопка «Удалить» в окне просмотра).
- Тумблер «включить / выключить приём» (глобальная настройка).

## Захват DOM (важно)

Скрипт снимает DOM не сразу, а **после полной загрузки страницы и «успокоения» динамики**: ждёт `window.load`, затем паузы в мутациях DOM (1.5 с) с потолком 12 с. Это нужно, потому что GetCourse догружает контент (AJAX, React-виджеты) уже после `DOMContentLoaded` — иначе в снимок попадал бы только ранний скелет. Скрипт кэшируется на 1 минуту: после правки скрипта уже открытые страницы получат новую версию в течение ~1 мин или при перезагрузке (жёсткое обновление — сразу).

---

## Известные ограничения

- Фильтр по `url` — точное совпадение, не подстрока.
- `dom` собирается без ограничения размера: крупные страницы могут раздувать таблицу. Платформа ingest принимает тело как минимум до 6 МБ.
- Снимок берётся один раз за загрузку страницы (на `window.load` + успокоение). Контент, появляющийся позже потолка 12 с, в снимок не попадёт.
- Абсолютный ingest-URL формируется из домена аккаунта через `ctx.account.url` (не хардкод).
- Домен в теге встройки (`s.chtm.khudoley.pro`) менять вручную при переносе.

### 2026-06-26 — Полнота DOM и удаление записей

- Захват DOM перенесён с `DOMContentLoaded` на `window.load` + успокоение динамики (MutationObserver, пауза 1.5 с, потолок 12 с) — раньше снимался ранний скелет, а не отрендеренная страница.
- Убран `keepalive` у отправки (его лимит тела ~64 КБ обрезал крупный DOM); проверено — ingest принимает до ≥6 МБ.
- Кэш встраиваемого скрипта снижен с 1 ч до 1 мин.
- Добавлено удаление записей: `POST /api/admin/diagnostics/delete` + кнопка «Удалить» в окне просмотра.
