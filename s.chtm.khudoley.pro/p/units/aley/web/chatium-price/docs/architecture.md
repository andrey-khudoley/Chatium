# Архитектура chatium-price

## Обзор

Проект — одностраничная витрина прайс-листа. Никакого состояния, БД (Heap),
клиентского интерактива и API нет. Всё рендерится на сервере в один HTML-ответ.

## Поток рендера

```
GET /  →  index.tsx (app.html)
          ├─ ctx.account.log(...)           # лог обращения
          ├─ читает TARIFFS из shared/pricing.ts
          ├─ собирает <TariffCard> на каждый тариф
          ├─ инлайнит SITE_CSS из shared/styles.ts в <style>
          └─ возвращает полный <html> (SSR через @app/html-jsx)
```

## Слои

- **index.tsx** — единственный роут `/`. Функциональный компонент `TariffCard`
  (поддержан `@app/html-jsx`: `block` может быть функцией `(props, ...children) => Child`).
- **shared/pricing.ts** — данные (тарифы, валюта, ссылки футера). Источник правды.
- **shared/styles.ts** — CSS-строка, инлайнится в `<style>`.

Вёрстка не хардкодит тарифы — только маппит `TARIFFS`. Правки контента — в `shared/`.

## Дизайн

Снят с chatium.ru/pricing: шрифт Inter (Google Fonts), белые карточки
(`border-radius: 16px`, мягкая тень, бордер `rgba(15,23,42,.08)`), зелёные галочки.
Акцентная карточка `card--featured` получает градиентное свечение через `::before/::after`.
Сетка адаптивна: 4 → 2 → 1 колонка.

## Почему без Vue

Страница полностью статическая. Пуре-SSR JSX проще, не тянет клиентский бандл
и не требует правил `// @shared` для клиента (см. `007-vue.md` — Vue нужен там,
где есть клиентские данные/интерактив).
