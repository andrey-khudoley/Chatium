# p/units/ipsen/kp — сайт КП «Ipsen · ИИ-агенты»

Одностраничный Chatium-сайт (SSR Vue) для клиента Ipsen: показывает архитектурную
концепцию, черновик коммерческого предложения и блок открытых вопросов. Мультиязычный
(RU/EN/FR, переключатель в шапке). Любой посетитель (в т.ч. анонимный) может оставить
ответ на вопрос без ввода имени — ответ сохраняется как «комментарий» и появляется
под вопросом.

Источник контента — проект second_brain `ipsen-ai-agents-discovery-e874ba`
(бриф, `architecture-ideas.md`, `kp-outline.md`).

## Структура

```
ipsen/kp/
├─ index.tsx               # app.html('/') — SSR: читает ответы из Heap, определяет initialLang, рендерит SitePage
├─ pages/SitePage.vue      # весь сайт: вкладки КП / Архитектура / Вопросы, переключатель RU/EN/FR, формы ответов
├─ api/answers/create.ts   # POST // @shared-route: приём ответа (requireAnyUser — аноним ok, без имени)
├─ tables/answers.table.ts # Heap-таблица ответов (комментарии), привязка по questionId
├─ shared/content.ts       # весь контент на 3 языках (CONTENT[Lang]), resolveLang(), LANGS  // @shared
├─ shared/styles.ts        # дизайн-CSS (светлая тема, красный акцент как в КП)
├─ config/routes.tsx       # PROJECT_ROOT + реестр путей (ссылок по URL в проекте нет)  // @shared
└─ манифесты: .dir.json .workspace.json package.json tsconfig.json jsx.d.ts vue-shim.d.ts
```

## Как это работает

1. `index.tsx` на сервере читает `Answers.findAll` (Heap доступен только на сервере),
   группирует ответы по `questionId`, определяет `initialLang` через `resolveLang(ctx.lang)`
   и передаёт всё SSR-пропсами в `SitePage.vue`.
2. Весь контент — `CONTENT: Record<Lang, SiteContent>` в `shared/content.ts` (RU/EN/FR).
   Переключение языка на клиенте мгновенное: `lang` — реактивный `ref`, весь текст —
   `computed` от него, без перезагрузки страницы и без обращения к платформенному `ctx.t()`
   (контент не текстовые ключи, а параллельные типизированные деревья по разделам).
3. Форма ответа — только текст, без поля имени. Отправляет данные не через `fetch`, а через
   `apiCreateAnswerRoute.run(ctx, { questionId, text })` (роут помечен `// @shared-route`).
4. `api/answers/create.ts` вызывает `requireAnyUser(ctx)` (анонимная запись разрешена),
   валидирует текст/вопрос и пишет запись в `Answers`. Имя не запрашивается: у реального
   пользователя берётся `displayName` автоматически, у анонима остаётся пустым — клиент
   в этом случае показывает только локализованный бейдж типа автора (гость/участник/бот),
   без пустой строки имени. Возвращает созданный ответ, который клиент добавляет в список
   без перезагрузки.
5. Раздел КП «Коммерческое предложение» — дефолтная вкладка. Пункты 1–4 заполнены,
   пункты 5–8 (Стоимость и сроки / Что требуется для старта / Передача / Следующий шаг)
   визуально заблокированы (`locked: true`, пунктирная рамка + иконка замка + подпись) —
   заполнятся после закрытия открытых вопросов.

## Данные

Таблица `Answers` (`t__ipsen-kp__answers__q7Kp2Za`): `questionId`, `text`,
`authorName` (пусто для анонимов), `authorUserId`, `authorType`, `createdAtMs`. Системные
`id/createdAt/updatedAt` добавляются Heap автоматически. Стабильные id вопросов —
11 штук (`q1`…`q15` с пропусками: q6/q8/q12/q13/q16-18 убраны по ходу правок) —
в `shared/content.ts`, одинаковые для всех трёх языков (ответы не сбрасываются
при смене языка).
