# Проект raffle07 (Ларина) — «Розыгрыш призов»

## Что это

Standalone-страница анимированного розыгрыша двух призов среди реального списка участников.
Порт дизайна Claude Design «Розыгрыш призов.dc.html» (проект `ae6d55cd-...`) на self-contained
vanilla-JS страницу — без DCLogic/support.js. Один серверный роут отдаёт готовый HTML.

URL: `/p/units/larina/raffle07`

## Где искать детали

- Техническая информация и changelog: `README.md`
- Точка входа (роут): `index.tsx` — `app.get('/', ctx => ctx.resp.html(PAGE_HTML))`
- Вся страница (разметка + стили + логика розыгрыша + данные): `pageHtml.ts` (`PAGE_HTML`)

## Ключевые факты

- Участники (42 чел.) захардкожены в `pageHtml.ts` → `USERS` (реальный список из user_export).
  email замаскированы (`xx**@**xx.xx`) — реальные адреса в исходник не попадают.
- Победители заданы жёстко: `WINNER_INDICES = [15, 25]` (0-based индексы в `USERS`).
  15 = Галина Берёзкина (#499919265) → 1 место, 25 = Ольга Самсонова (#502157186) → 2 место.
  Смена победителей = правка двух чисел в `WINNER_INDICES`.
- Призы: `PRIZES` (1/2 место) + `CONSOLATION` (2000 бонусных рублей всем).
- Старт по кнопке: начальная фаза `idle` — оверлей (z-80) с кнопкой «Запустить розыгрыш»
  (`.rz-start-btn`, hover/active + sheen). Клик → `start()`: флаг `starting` (fade `idleOut`) →
  `phase:'intro'` → через 700 мс `runDraw(0)`. Авто-старта нет.
- Анимация полностью клиентская, двухэтапная: idle → intro → draw (этап 1 `scanCol`×3 — отбор
  финалиста из каждого столбца одновременно; этап 2 `scanSeq`/`runFinal` — перебор трёх финалистов)
  → reveal → results. Состояние: `starting`, `finalists`, `finalRound`, `colHi[3]`. Конфетти на `<canvas>`.
- Мобильная адаптация — на классах `rz-*` (`@media 900px/560px`): сетка участников всегда 3 столбца
  (нужно для параллельной анимации), остальное ужимается/перестраивается.

## Правила

- Не хардкодить URL. Страница self-contained, серверных данных нет (heap/таблицы не используются).
- `pageHtml.ts` — единственный источник разметки; править визуал/логику только там.
