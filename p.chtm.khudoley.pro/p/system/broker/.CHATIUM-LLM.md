# Проект broker

## Что это

Глобальный брокер событий уровня аккаунта (публикация/подписка модулей на события, доставка через pull-модель). Это dev-копия (окружение stage); prod-копия — `p/system/broker`, появится при переносе через `/to-prod`.

## Где искать детали

- Источник истины: `docs/spec/spec.md` — код меняется только после того, как изменение описано здесь (spec-as-source, §0 спеки)
- Карта спеки: `docs/spec/spec-map.md`
- Техническая информация и changelog: `README.md`
- Архитектура: `docs/architecture.md`
- API: `docs/api.md`
- Данные: `docs/data.md`
- Решения: `docs/ADR/` (0001–0015)
- История диалогов: `docs/LLM/`

## Тесты

- Интерактивно: `web/tests`
- Гейт (для CI/приёмки): `web/tests/ai` — JSON-эндпоинт, зелёный прогон = `summary.success === true`; поддерживает `?category=` для прогона одной категории
- Реестр тестов: `shared/tests/test-definitions.ts`; раннер и наборы — `lib/tests/`

## Ключевые правила

- **spec-as-source (§0 спеки).** Любое изменение поведения, контрактов, данных или структуры — сначала правка `docs/spec/spec.md`, потом код.
- **Пары таблиц stage/prod.** Каждая Heap-таблица существует в двух версиях (`*_stage_*` / `*_prod_*`); наружу отдаётся только таблица текущего окружения через `IS_PROD`-селектор в `config/env.ts`.
- Не хардкодить URL — только `config/routes.tsx`.
