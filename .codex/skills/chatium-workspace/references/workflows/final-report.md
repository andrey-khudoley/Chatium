> This is a Codex workflow reference. The old stage `.cursor` directory was removed after migration.

Prepare the final task report after implementation and verification.
Use it at the end of a pipeline or whenever the user asks for a compact status/result summary.

## Report Template

1. **Задача:** исходная формулировка в 1-2 предложениях.
2. **Acceptance criteria:** checklist with done / partial / not done status.
3. **Что сделано:** changed files and the purpose of each change; new pages, API routes, tables, jobs, or docs.
4. **Что проверено:** commands, review/checker results, test scenarios, and any manual verification.
5. **Что осталось открытым:** limitations, residual risks, manual checks, or follow-up work.
6. **Документация:** whether `README.md`, `.CHATIUM-LLM.md`, `docs/*`, or `docs/LLM/*` were updated or intentionally left unchanged.

## Codex Notes

- Keep the report concise and user-facing.
- Mention commands that could not be run and why.
- Do not include large code snippets.
- Use absolute clickable file links when referring to local files in the final response.
